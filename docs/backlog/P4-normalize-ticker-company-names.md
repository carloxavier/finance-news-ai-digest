# P4 — Normalize ticker → company name via a dedicated tickers table

**Filed**: 2026-04-18
**Status**: open

## Summary

Company names for tickers currently live only inside `analyst_data`
jsonb, as free-text strings generated by the LLM at seed time. Every
time the seeder writes an article with tickers, it independently
populates `companyName` per ticker entry. This is unreliable:

- **No validation**: a typo or inconsistent format gets written
  silently.
- **No source of truth**: two articles mentioning AAPL could show
  different company names depending on what the LLM generated.
- **No scalability**: any future ticker attribute (sector,
  exchange, logo URL) would have the same reliability problem at
  a larger surface.

The P3 ticker+company-name rendering fix makes the unreliability
visible in the UI. That was a symptom fix; this is the structural
one.

## Target state

A `tickers` reference table as the canonical registry:

```sql
CREATE TABLE tickers (
  ticker text PRIMARY KEY,
  company_name text NOT NULL,
  sector text,
  exchange text,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);
```

Seeder and frontend resolve `ticker → company_name` through this
table, not through free-text in jsonb.

## Migration path

1. Create `tickers` table.
2. Backfill: extract every distinct `(ticker, companyName)` pair
   from current `analyst_data` jsonb; dedupe; insert. Manual
   review for conflicts.
3. Update seeder skill v3: when writing `analyst_data`, query
   `SELECT company_name FROM tickers WHERE ticker = $1`. If a
   ticker isn't registered, fail loudly — forces a human to add
   the row before seeding continues.
4. Frontend: keep reading `companyName` from denormalized jsonb
   for now (simpler, avoids extra JOIN per article); migrate to
   JOIN at query time when the cost matters.
5. Future: wire auto-population from Finnhub/IEX/Yahoo when a
   ticker is first encountered.

## Scope

- Migration creating the `tickers` table.
- Backfill script for existing `(ticker, companyName)` pairs.
- Seeder skill v3 update.
- Document in `docs/data-model/reference-tables.md`.

## Acceptance

- `tickers` table exists and is populated from current data.
- Seeder skill v3 refuses to seed articles referencing unknown
  tickers.
- No silent-typo entries persist in any `analyst_data` jsonb
  after backfill.

## Priority

P4 — current free-text approach works at 3 v2 articles; gets worse
as corpus grows. Do before rebuilding the v2 corpus to hundreds of
articles.
